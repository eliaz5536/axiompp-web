Set theory studies sets, which are collections of objects. It is a fundamental part of mathematics and provides the basis for various mathematical concepts and structures.

Set theory includes union, intersection, and difference of sets, as well as the concepts of subsets, power sets, and cardinality. It also explores relations, functions, and equivalence relations.

Furthermore, it is the foundation of mathematical logic and is used in various fields such as computer science, statistics, and probability theory.

It is then further used in the study of relations, functions, and equivalence relations, which are essential concepts in mathematics and computer science.

It is expanded by Bertrand Russell and Alfred North Whitehead in their work "Principia Mathematica," which aimed to provide a solid foundation for mathematics based on set theory. 

It is also to avoid paradoxes such as Russell's paradox, which arises from considering the set of all sets that do not contain themselves.

This is essential for theorem proving and formalizing mathematical concepts, as it provides a rigorous framework for reasoning about collections of objects and their relationships, using intuitionistic and formal methods.


# Set 
Set is a collection of distinct objects, considered as an object in its own right. Sets are fundamental objects in mathematics. For example, the numbers 1, 2, and 3 are distinct objects when considered separately, but when they are considered collectively as the set {1, 2, 3}, they form a single object.

```cpp
Set<int> primes{2, 3, 5};
return primes;
```

# Relation
Relation is a connection or association between elements of two or more sets. It can be represented as a set of ordered pairs, where each pair consists of an element from one set and an element from another set. Relations can be classified into different types, such as reflexive, symmetric, transitive, and antisymmetric relations.

```cpp
int a = 5; // positive integer
int b = -3; // negative integer
int c = 0; // zero
return 0;
```

# Union
The union of two sets A and B is the set of all elements that are in either A or B (or both).

```cpp
Set<int> primes{2, 3, 5};
Set<int> composites{4, 6, 8};
Set<int> unionSet = primes.unionWith(composites);

return unionSet;
```

# Partial Order
Partial Order is a binary relation over a set that describes how elements can be compared to each other. A partial order is reflexive, antisymmetric, and transitive. It allows for the comparison of some elements while leaving others incomparable.

```cpp
PartialOrder<int> divisibilityOrder;
divisibilityOrder.add(1, 1);
divisibilityOrder.add(2, 2);
divisibilityOrder.add(4, 4);
divisibilityOrder.add(1, 2);
divisibilityOrder.add(2, 4);
divisibilityOrder.add(1, 4);
std::cout << "Partial order less-or-equal(1,4): " << divisibilityOrder.lessEqual(1, 4) << "\n";
std::cout << "Partial order comparable(2,4): " << divisibilityOrder.comparable(2, 4) << "\n";

return 0;
```

# Ordered Pair
Ordered Pair is a pair of elements in which the order of the elements matters. It is denoted as (a, b), where 'a' is the first element and 'b' is the second element. Ordered pairs are used to define relations and functions.

```cpp
OrderedPair<int, std::string> point(7, "seven");
std::cout << "Ordered pair: " << point << "\n";

return 0;
```

# Function
Function is a relation between a set of inputs and a set of possible outputs, where each input is related to exactly one output. Functions can be represented as mappings from one set (the domain) to another set (the codomain).

In set Theory, a function f from set A to set B is denoted as f: A → B, where for each element a in A, there exists a unique element b in B such that f(a) = b.

Function in set theory is used to describe relationships between sets and to define mathematical operations. Functions can be classified into different types, such as injective (one-to-one), surjective (onto), and bijective (both one-to-one and onto) functions.

```cpp
Function<int, std::string> nameByNumber;
nameByNumber.add(1, "one");
nameByNumber.add(2, "two");
nameByNumber.add(3, "three");
std::cout << "Function cardinality: " << nameByNumber.cardinality() << "\n";
std::cout << "Function apply(2): " << nameByNumber.apply(2) << "\n";
std::cout << "Function injective: " << nameByNumber.isInjective() << "\n";
std::cout << "Function bijective over {one,two,three}: " << nameByNumber.isBijective(Set<std::string>{"one", "two", "three"}) << "\n";
```

# Equivalence Relation
Equivalence relation is a binary relation that satisfies three properties: reflexivity, symmetry, and transitivity. It partitions a set into equivalence classes, where each class contains elements that are equivalent to each other under the relation.

```cpp
EquivalenceRelation<int> congruence;
congruence.add(1, 1);
congruence.add(2, 2);
congruence.add(3, 3);
congruence.add(1, 2);
congruence.add(2, 1);
std::cout << "Equivalence relation reflexive: " << std::boolalpha << congruence.isReflexive(Set<int>{1, 2, 3}) << "\n";
std::cout << "Equivalence class of 1: " << congruence.equivalenceClass(1) << "\n";
```

# Cartesian Product
Cartesian Product is the set of all ordered pairs formed by taking one element from each of two sets. If A and B are two sets, their Cartesian product is denoted as A × B and consists of all possible ordered pairs (a, b) where a ∈ A and b ∈ B.

In set theory, the Cartesian product is used to define relations and functions, as well as to construct higher-dimensional spaces. It is a fundamental concept in mathematics and has applications in various fields, including computer science, physics, and engineering.

```cpp
EquivalenceRelation<int> congruence;
congruence.add(1, 1);
congruence.add(2, 2);
congruence.add(3, 3);
congruence.add(1, 2);
congruence.add(2, 1);
std::cout << "Equivalence relation reflexive: " << std::boolalpha << congruence.isReflexive(Set<int>{1, 2, 3}) << "\n";
std::cout << "Equivalence class of 1: " << congruence.equivalenceClass(1) << "\n";`
```
